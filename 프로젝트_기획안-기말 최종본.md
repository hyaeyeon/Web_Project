# [프로젝트기획안] 전공 선택 도우미 (Major Choice Helper) 웹 서비스 구축

## 1. 프로젝트 개요
### 1.1 프로젝트 명
- **국문**: 전공 선택 도우미
- **영문**: Major Choice Helper

### 1.2 프로젝트 목적
- IT학부 학부생으로서 최근에 컴퓨터공학과와 융합보안학과 중에서 어느 과를 전공으로 선택해야 할지 고민을 하게 됨. 
- 진로와 전공 선택에 어려움을 겪는 학생들에게 직관적인 테스트와 정보를 제공하여 의사결정을 지원함.
- 복잡한 회원가입 절차 없이(또는 간소화하여) 누구나 쉽게 접근 가능한 웹 기반 플랫폼 구축.
- 단순 정보 나열이 아닌, 비교 분석 및 로드맵 시각화를 통해 구체적인 진로 설계를 도움.
- IT학부 학부생, 컴퓨터공학과와 융합보안학과에 대해서 알고 싶은 사람들을 대상으로 함.

### 1.3 개발 목표
- **접근성**: 별도의 설치 없이 웹 브라우저에서 즉시 실행 가능한 환경 제공.
- **직관성**: 사용자 친화적인 UI/UX를 통해 정보 습득의 피로도 최소화.
- **확장성**: 추후 데이터베이스 연동 및 커뮤니티 기능 확장을 고려한 모듈형 구조 설계.

## 2. 서비스 구성 및 기능 명세서
### 2.1 주요 기능 (Key Features)

#### A. 전공 적성 검사 (Aptitude Test)
- **기능**: MBTI 스타일의 문항을 통해 사용자의 성향을 분석하고 적합한 전공 계열(공학, 인문, 예술 등)을 추천.
- **구현**: `test.html`, `test.js`
- **로직**: 각 문항 선택 시 점수 누적 -> 최종 점수에 따른 결과 페이지 매핑.

#### B. 전공 비교 분석 (Major Comparison)
- **기능**: 사용자가 관심 있는 두 가지 전공을 선택하여 커리큘럼, 진로 전망, 장단점을 한눈에 비교.
- **구현**: `compare.html`, `compare.js`
- **UI**: 좌우 분할 레이아웃을 통한 직관적인 비교 인터페이스 제공.

#### C. 학습 로드맵 (Learning Roadmap)
- **기능**: 특정 전공 선택 시 1학년부터 4학년까지의 주요 교과목 및 필요 역량을 시각적 타임라인으로 제시.
- **구현**: `roadmap.html`, `roadmap.js`
- **데이터**: 학년별 이수 체계도 시각화.

#### D. FAQ 및 정보 (FAQ)
- **기능**: 전공 선택과 관련된 자주 묻는 질문과 답변 제공.
- **구현**: `faq.html`, `faq.js`
- **인터랙션**: 아코디언(Accordion) UI를 활용한 질문/답변 펼치기 기능.

#### E. 사용자 관리 (User System)
- **기능**: 회원가입 및 로그인 기능을 통해 사용자 개인화(추후 확장 대비).
- **구현**: `login.html`, `signup.html`, `login.js`, `signup.js`
- **데이터 저장**: `localStorage`를 활용한 클라이언트 사이드 세션 관리 (프로토타입 단계).

### 2.2 정보 구조 (Information Architecture)
```mermaid
graph TD
    A[메인 페이지 (Index)] --> B[로그인/회원가입]
```

## 3. 기술 스택 (Tech Stack)
### 3.1 프론트엔드 (Frontend)
- **HTML5**: 시맨틱 태그를 활용한 웹 표준 준수 및 구조화.
- **CSS3**: Flexbox 및 Grid 레이아웃을 활용한 반응형 디자인, CSS Variables를 이용한 테마 관리.
- **JavaScript (ES6+)**:
  - DOM 조작을 통한 동적 콘텐츠 렌더링.
  - `localStorage` API를 활용한 데이터 영속성 처리 (회원 정보, 테스트 결과 등).
  - 외부 라이브러리 의존성 없는 순수 바닐라 JS(Vanilla JS) 구현으로 가벼운 성능 유지.

### 3.2 개발 환경 및 도구
- **IDE**: Visual Studio Code
- **Version Control**: Git (형상 관리)
- **AI Assistance**: GitHub Copilot & Vibe Coding (코드 생성 및 리팩토링 지원)

---

## 4. 기대 효과
- **사용자 측면**: 막연했던 전공 선택 과정을 구체적인 데이터와 시각 자료를 통해 명확화할 수 있음.
- **기술적 측면**: 프레임워크 없이 순수 웹 기술만으로 SPA(Single Page Application)와 유사한 사용자 경험을 구현하여 웹 기초 역량 강화.

## 5. 향후 발전 과제
- **백엔드 도입**: 현재 `localStorage`에 의존하는 데이터를 서버 DB(Firebase, Supabase 등)로 이관하여 데이터 안정성 확보.
- **커뮤니티 활성화**: 현직자 또는 선배 멘토링 기능 추가.
- **데이터 고도화**: 대학 알리미 등 공공 API를 연동하여 실시간 취업률 및 등록금 정보 제공.
