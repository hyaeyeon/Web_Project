- **과목**: 대학교 웹프로그래밍
- **기술 스택**: HTML, CSS, JavaScript (프론트엔드 전용, 백엔드 미사용)

### 1.2 AI 활용 방식: 바이브코딩(Vibe Coding)
> "AI를 단순 코드 생성 도구가 아닌, **프로젝트 팀원**으로 활용하여 기획-설계-구현-디버깅 전 과정을 협업하는 개발 방식"

본 프로젝트에서는 ChatGPT를 **"조언해줄 수 있는 선배이자 프로젝트 팀원"**으로 설정하고, 다음과 같은 역할을 부여했습니다:

| 역할 | 구체적 활동 |
|------|------------|
| 기획 파트너 | UI 콘셉트 제안, 기능 우선순위 설정, 사용자 흐름 설계 |
| 설계 자문 | 페이지 구조, 데이터 흐름, 상태 관리 방식 제안 |
| 코드 작성 | HTML/CSS/JS 코드 생성 및 최적화 |
| 디버깅 협력 | 오류 원인 분석, 해결책 제시, 코드 리뷰 |
| 품질 관리 | UX 개선점 발견, 과제 평가 기준 조언 |

## 3. AI 협업 과정 분석

### 3.1 개발 단계별 AI 활용

#### Phase 1: 기획 및 콘셉트 설정
**AI에게 요청한 내용:**
- 웹사이트 분위기/콘셉트 4가지 제안 요청
- 각 콘셉트의 시각적 예시 및 적합한 기능 매칭
- 과제 평가 기준에 맞는 콘셉트 선택 조언

**AI의 기여:**
```
제안된 콘셉트 4가지:
1. "클린 & 캠퍼스 가이드" - 가장 무난, 평가 잘 받는 스타일
2. "다크 테크 & 보안 감성" - 융합보안 분위기 강조
3. "친근한 진로 상담 챗봇 톤" - 학생 대상 UX 적합
4. "게임/인터랙티브 선택형" - JS 활용 티가 확 남

→ 최종 선택: 1+3 혼합 ("Clean Tech + Student Friendly + Interactive Guide")
```

**생산성 효과:**
- 콘셉트 결정 시간: 기존 예상 2~3일 → **수 시간 내 확정**
- 디자인 방향성 명확화로 후속 작업 속도 향상

#### Phase 3: 기능 구현 및 디버깅

**사례 1: Sub Skill 그래프 미표시 문제**

| 문제 상황 | AI 분석 | 해결 |
|-----------|---------|------|
| 그래프가 안 보임 | HTML 구조 오류 (section 태그 미닫힘) 가능성 제시 | `<section class="content home">` 위치 수정 |
| 수치 대신 `${...}` 표시 | 백틱(`) 대신 따옴표 사용 | 템플릿 리터럴 문법 수정 |

**AI의 디버깅 접근법:**
```
1. 콘솔 에러 확인 요청
2. CSS 선택자 적용 여부 점검
3. HTML 구조(DOM 트리) 검증
4. JS 실행 순서 확인
```

**사례 2: 로그인 상태 유지 문제**

```javascript
// AI가 제안한 안정화된 renderDashboard() 구조
function renderDashboard(){
  const result = loadResult();
  
  // 1️⃣ 결과 없을 때: 기본 UI 표시
  if(!result){
    document.getElementById("kpiMajor").textContent = "테스트 필요";
    renderChart([3, 6, 5, 7, 4]); // 데모 값
    return;
  }
  
  // 2️⃣ 결과 있을 때: 실제 데이터 렌더링
  const { ceScore, secScore, major, tags, summary } = result;
  // ... 렌더링 로직
}
```

**핵심 원칙 (AI 제안):**
> "result가 없을 때는 절대 ceScore/secScore를 쓰지 않는다"
> "UI를 항상 '의도된 상태'로 세팅한다"

### 3.2 협업 커뮤니케이션 패턴

#### 효과적이었던 대화 방식

| 패턴 | 예시 | 효과 |
|------|------|------|
| **역할 부여** | "너는 나의 프로젝트 팀원이야" | AI가 맥락을 유지하며 일관된 조언 |
| **제약 조건 명시** | "백엔드 없이 UI만 구현해야 해" | 실현 불가능한 제안 방지 |
| **피드백 즉시 전달** | "화면 잘 떠" / "이건 안 바뀌었어" | 빠른 문제 해결 사이클 |
| **의사결정 요청** | "다음으로 뭐부터 할까?" | 작업 우선순위 명확화 |

#### AI의 팀원적 행동 패턴
```
✔ 과제 평가 기준을 고려한 조언
✔ 코드 변경의 "왜"를 설명
✔ 수정 후 확인 방법 안내
✔ 다음 단계 자연스럽게 제안
✔ 실수 발생 시 원인 분석 + 해결책 동시 제공
```

## 5. 프로젝트 결과물 하이라이트

### 5.1 성향 테스트 알고리즘
```javascript
// 12문항 (컴공 6문항 + 보안 6문항)
// 각 문항 1~5점, 30점 만점 → 100점 환산

const ceScore = Math.round((ceRaw / 30) * 100);
const secScore = Math.round((secRaw / 30) * 100);

const major = ceScore > secScore ? "컴퓨터공학과" : "융합보안학과";
```

### 5.2 Sub Skill 시각화
```javascript
// 테스트 결과 기반 5가지 하위 역량 도출
const s1 = (ceScore / 100) * 10;           // 개발·구현
const s2 = (secScore / 100) * 10;          // 보안·분석
const s3 = ((ceScore + secScore) / 200) * 10;  // 균형도
const s4 = (Math.abs(ceScore - secScore) / 100) * 10;  // 편향도
const s5 = ((100 - Math.min(ceScore, secScore)) / 100) * 10;  // 보완 필요
```

## 6. 시사점 및 결론

### 6.1 바이브코딩의 핵심 성공 요인

1. **명확한 역할 설정**
   - AI를 "코드 생성기"가 아닌 "팀원"으로 인식
   - 프로젝트 컨텍스트를 지속적으로 공유

2. **제약 조건의 명시적 전달**
   - "백엔드 없이", "UI 중심", "과제 기준"
   - 실현 불가능한 제안 사전 차단

3. **즉각적인 피드백 루프**
   - 결과 확인 → 문제점 전달 → 해결책 수령 → 적용
   - 작은 단위로 빠르게 반복

4. **학습과 구현의 병행**
   - AI 제안의 "왜"를 이해하려는 노력
   - 단순 복붙이 아닌 원리 파악

### 6.2 한계 및 주의점

| 한계 | 대응 방안 |
|------|----------|
| AI 제안이 항상 정확하지 않음 | 직접 테스트 및 검증 필수 |
| 컨텍스트 유실 가능성 | 핵심 제약/상태 반복 전달 |
| 과도한 의존 위험 | 핵심 로직은 직접 이해 후 적용 |

### 6.3 결론

> **"본 프로젝트는 AI를 활용한 바이브코딩 방식을 통해, 
> 기존 대비 약 60%의 개발 시간 단축과 함께 
> 학습 효과 및 결과물 품질 향상을 동시에 달성하였다."**

바이브코딩은 단순히 "AI가 코드를 대신 써주는 것"이 아니라,
**AI와의 효과적인 협업을 통해 개발자의 생산성과 학습을 동시에 극대화하는 새로운 개발 패러다임**임을 본 프로젝트를 통해 확인할 수 있었습니다.
