학년별 로드맵

How to use

학년(또는 단계)별로 접기/펼치기를 하며 확인하세요. 한세대학교의 컴퓨터공학과와 융합보안학과를 기준으로 정리되어 있는 로드맵입니다.

💻 컴퓨터공학과 로드맵

시스템·소프트웨어 전반의 구조를 이해하고, 구현 역량(코드/프로젝트/협업)을 강화하는 흐름입니다.

핵심 키워드 · 기초 프로그래밍 · 논리적 사고 · 전산 기초

  • 진로탐색: IT학부 전공탐색 세미나
  • 프로그래밍 기초(C/Python): 문법, 함수, 자료형, 디버깅 습관
  • 문제 해결 사고: 작은 문제로 쪼개기, 조건/반복 구조 설계
  • 컴퓨터 구조·운영체제 맛보기: 컴퓨터개론, CPU/메모리/프로세스 기본 개념
  • 추천 준비: Git/GitHub로 버전관리, 간단한 콘솔 미니프로그램 2~3개

핵심 키워드 · 자료구조 · 객체지향 · DB · 네트워크 기초

  • 자료구조 및 알고리즘: 스택/큐/리스트/트리/해시의 쓰임과 구현 감각
  • 객체지향(Java/C++): 클래스 설계, 캡슐화/상속/다형성 이해
  • 데이터베이스(SQL), 데이터 통신: 정규화 개념, CRUD, 간단한 스키마 설계
  • 네트워크 기초: TCP/IP, HTTP, 요청-응답 흐름 이해
  • 인공지능 기초, 인공지능 프로그래밍
  • 추천 준비: 작은 CRUD 웹/콘솔 프로젝트 1개, SQL 실습(SELECT/JOIN)

핵심 키워드 · 웹/서버 · 알고리즘 심화 · 협업

  • 웹/서버: 프레임워크 기반 개발 흐름(라우팅, API, 인증 개념)
  • 알고리즘: 시간복잡도/공간복잡도, 탐색/정렬/DP 기초
  • 협업: 이슈 관리, 브랜치 전략, 코드 리뷰 경험
  • 캡스톤디자인
  • 추천 준비: 팀 프로젝트 1회 이상, 해커톤/인턴십/동아리 활동

핵심 키워드 · 포트폴리오 · 실무 역량 · 면접 준비

  • 졸업 프로젝트: 기획→구현→발표까지 문서화/시연 완성
  • 취창업세미나
  • 관심 분야 특화: 백엔드/프론트/데이터/AI 등 선택과 집중
  • 면접 대비: 프로젝트 설명, CS 기초(네트워크/OS/DB) 정리
  • 추천 준비: 포트폴리오 웹페이지/README 정비, 코딩테스트 루틴
🔐 융합보안학과 로드맵

시스템/네트워크의 동작 원리를 이해하고, 위협·취약점·대응의 관점을 단계적으로 확장하는 흐름입니다.

핵심 키워드 · 보안 개요 · 네트워크 기초 · 윤리

  • 전공탐색: IT학부 전공탐색 세미나
  • 융합보안개론: 기밀성/무결성/가용성(CIA)와 위협 모델
  • 네트워크 기초: IP/포트/프로토콜, HTTP 요청-응답 이해
  • 프로그래밍 기초: 입력 검증, 파일 처리 등 기본 구현 역량
  • 추천 준비: 리눅스 기본 명령어, 보안 용어 노트 정리

핵심 키워드 · 시스템 보안 · 네트워크 보안 · 암호 기초

  • 시스템 보안: 계정/권한, 프로세스/메모리 기본 개념
  • 네트워크 보안: 방화벽/IDS/IPS 개념, 트래픽 흐름 이해
  • 암호 기초, 해킹과 보안: 해시/대칭/비대칭 개념과 사용 시나리오
  • 리눅스 시스템
  • 사이버범죄심리, 보안법제도 기초
  • 추천 준비: 로그를 읽는 습관, 간단한 보안 실습 환경 경험(개념 중심)

핵심 키워드 · 침해사고 대응 · 포렌식 기초 · 관제

  • 침해사고 흐름: 탐지→분석→차단→복구→재발 방지 관점
  • 포렌식 기초: 증거 보존, 타임라인/로그 기반 분석 개념
  • 보안 관제: 알림 분석, 오탐/정탐 구분의 기본 프레임
  • 과학기술범죄과학, 시큐어 코딩
  • 추천 준비: CTF(초급)로 사고력 훈련, 팀 프로젝트로 보고서 작성 경험

핵심 키워드 · 전문 분야 선택 · 포트폴리오 · 자격 준비

  • 세부 분야 선택: 관제/SOC, 모의해킹, 포렌식, 컨설팅 등
  • 졸업 프로젝트: 보안 요구사항을 반영한 설계/구현/발표
  • 자격 준비: 학부 수준에서 가능한 범위의 목표 설정
  • 추천 준비: 케이스 스터디(사고 분석) 정리, 발표 자료/보고서 템플릿 구축
Tip
  • 컴퓨터공학과는 “구현 + 협업 + 구조 이해”가 핵심입니다. 프로젝트 결과물을 꾸준히 쌓으세요.
  • 융합보안학과는 “원리 이해 + 분석 관점 + 대응 흐름”이 핵심입니다. 로그/사례 기반으로 정리해보세요.
  • 로드맵은 “정답”이 아니라 “방향”입니다. 본인의 흥미와 강점을 기준으로 조정하면 가장 좋습니다.
