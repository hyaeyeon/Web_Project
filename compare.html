컴퓨터공학과 (Computer Engineering / CS 중심)
개발·구현
알고리즘
시스템/DB
소프트웨어 설계
  • 프로그래밍 기초: 문법/코드 구조/디버깅 습관
  • 자료구조·알고리즘: 효율적으로 문제를 푸는 사고(시간/공간 복잡도)
  • 컴퓨터 시스템: 운영체제, 메모리, 프로세스, 스케줄링 같은 “컴퓨터 내부 동작”
  • DB/데이터 모델링: 관계형 모델, SQL, 트랜잭션 개념(정합성)
  • 소프트웨어 공학: 요구사항→설계→구현→테스트→협업(Git) 과정
  • 구현력: 기능을 코드로 “완성”시키는 능력
  • 문제 해결력: 요구사항을 작은 단위로 쪼개고 로직을 설계하는 능력
  • 확장·유지보수: 구조를 깔끔하게 만들어 나중에 고치기 쉬운 코드 작성
  • 협업 능력: 버전관리(Git), 이슈/PR 기반 협업 흐름 이해
  • 회원가입/로그인 UI(프론트) + CRUD 설계(개념)
  • DB 설계(ERD) → SQL로 테이블 구성 → 데이터 처리
  • 간단한 웹 서비스(소개/테스트/대시보드) 구현
  • 알고리즘 문제 풀이/코딩테스트 대비
  • 백엔드: API 설계, DB 연동, 서버 로직 구성
  • 프론트엔드: UI/UX 구현, 상태 관리, 성능 최적화
  • 모바일/앱: 앱 화면+기능 구현, 디바이스 특성 대응
  • 데이터/ML 엔지니어링: 데이터 파이프라인, 모델 서빙(상위 단계)
융합보안학과 (Security / Assurance 중심)
분석·탐지
네트워크/시스템
취약점/위협
정책·리스크
  • 네트워크 기초: IP/TCP/HTTP 같은 통신 구조(웹이 “어떻게 오가는지”)
  • 시스템/운영체제: 권한, 계정, 프로세스, 로그, 파일 시스템 이해
  • 보안 원리: 인증/인가, 암호 기초 개념, 접근통제, 보안 설계
  • 취약점 이해: 웹 취약점(예: 입력 검증, 권한 문제) 같은 구조적 약점
  • 거버넌스/정책: 리스크 관리, 준수(컴플라이언스), 보안 운영 관점
  • 분석력: “무슨 일이 일어났는지” 로그/흐름으로 재구성
  • 예방 설계: 시스템을 만들 때 보안 이슈가 생기지 않게 기준을 세움
  • 리스크 관점: 위험도를 평가하고 우선순위를 정하는 능력
  • 커뮤니케이션: 기술을 비기술자에게 설명하고 보고서로 정리
  • 간단한 로그 분석 시나리오(이상 징후 탐지) 정리
  • 웹 취약점 체크리스트 기반 점검(개념/시뮬레이션)
  • 보안 사고 대응 절차(탐지→분석→조치→재발방지) 문서화
  • 보안 정책/가이드라인 작성(사용자 권한/비밀번호 정책 등)
  • 보안관제(SOC): 경보 분석, 위협 탐지, 대응 협업
  • 침해사고 분석(IR): 사고 원인/범위 분석, 재발 방지 대책
  • 보안 컨설팅: 진단/개선안, 정책/프로세스 설계
  • 보안 엔지니어: 보안 시스템 운영, 설정/모니터링/개선
공통으로 겹치는 영역 (둘 다 도움이 되는 기반)
공통 기반
  • 프로그래밍 기초(문법/디버깅 습관)
  • 웹/네트워크 기본(HTTP, 요청/응답 흐름)
  • DB 기초(SQL, 데이터 모델링 개념)
  • 협업(Git, 문서화)
선택 기준 힌트
  • “서비스를 만들고 싶다” → 컴공 성향이 더 강할 가능성
  • “사고를 막고 원인을 파고든다” → 보안 성향이 더 강할 가능성
  • 둘 다 끌리면 “직무(백엔드/IR/컨설팅)”를 먼저 정해도 좋음
